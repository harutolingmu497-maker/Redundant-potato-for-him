このフォルダには簡単な GUI ダイアログ（Web モーダル）のサンプルがあります。

ファイル:
- `web_modal.html` : シンプルなモーダルダイアログの静的デモ。ブラウザで開いて動作を確認できます。

使い方:
1. `examples/web_modal.html` をブラウザで開きます（ダブルクリックで開けます）。
2. 「削除ダイアログを開く」ボタンを押してダイアログを表示します。
3. Tab / Shift+Tab / Esc でキーボード操作を確認できます。

注意:
- 実運用ではフォーカストラップやスクリーンリーダー向けのラベル強化（aria-describedby など）を追加してください。

アクセシビリティチェックリスト
---------------------------
以下はこのサンプルを手動/自動で確認するための簡易チェックリストです。プロダクション導入時の最低ラインとして利用してください。

手動チェック（ブラウザでの確認）
- キーボード操作
	- Tab / Shift+Tab でフォーカスがダイアログ内に閉じ込められる（フォーカストラップ）。
	- Esc でダイアログが閉じる。
	- ダイアログを閉じた後、トリガーボタンにフォーカスが戻る。
- スクリーンリーダー
	- タイトル（`aria-labelledby`）と説明（`aria-describedby`）が読まれる。 
	- ダイアログ表示中に背景要素が読み上げられない（`aria-hidden` で隠れている）。
- フォーカス表示
	- ボタンや入力にフォーカス時の視覚的指示がある（アウトライン等）。
- ラベルとコントラスト
	- ボタンやラベルが具体的で誤解を招かない文言か（例: 「削除」や「キャンセル」）。
	- 背景/文字のコントラストが WCAG の推奨値を満たす（最小 AA）。

自動チェック（推奨ツール）
- Lighthouse（Chrome DevTools に内蔵）: Accessibility スコアと簡易改善案を確認。
- axe-core（ブラウザ拡張 or npm）: 具体的なアクセシビリティ違反を検出。
- pa11y / pa11y-ci: CI に組み込んで回帰を防止。

簡易コマンド例（PowerShell / Windows）
（pa11y を使う例。事前に Node.js と npm が必要です）
```pwsh
# pa11y を global にインストール（必要な場合）
npm install -g pa11y

# ローカルで簡易サーバを立てて検査する（例: Python の http.server）
# PowerShell で examples ディレクトリに移動して実行:
cd examples
python -m http.server 8000

# 別のターミナルで pa11y を実行
pa11y http://localhost:8000/web_modal.html
```

テスト/期待結果の例
- キーボード: Tab を押し続けた場合、フォーカスがダイアログ内をループする。
- スクリーンリーダー: ダイアログ開口時にタイトルと説明が読み上げられ、背景は読まれない。
- 自動ツール: 重大な a11y エラー（コントラスト不足、aria 属性の欠落など）が検出されない。
